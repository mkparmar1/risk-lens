# RiskLens - AI Freelance Risk Analyzer

RiskLens is an AI-powered risk analysis tool for freelancers. It allows users to input project details (scope, budget, timeline) and upload contracts/documents to receive a comprehensive risk assessment, including a probability score, strategic recommendations, and suggested contract clauses.

## üåü Features

### 1. **AI Risk Analysis Engine**
- **Core Functionality**: Wizard-based interface to collect project data.
- **Input**:
  - **Basic Info**: Project Title, Client Name, Description.
  - **Documents**: File upload interface. **Supports PDF and DOCX analysis.** The system extracts text from uploaded files to provide context-aware risk assessment.
  - **Constraints**: Budget, Currency, Timeline.
  - **Communication**: Open text field for client emails/notes.
- **Output**:
  - **Risk Score**: 0-100 probability.
  - **Risk Level**: Low/Medium/High.
  - **Recommendations**: Accept/Negotiate/Reject advice.
  - **Contract Clauses**: Specific legal clauses generated by AI to mitigate risks.
- **Intelligence**:
  - **Client Memory**: Checks if the client has a history of high-risk projects in the database.

### 2. **User Dashboard**
- **Statistics**: Displays total projects, average risk score, and high-risk project count.
- **Project History**: List of past analyses with status (Draft, Completed, Failed).
- **Search**: Filter projects by name.

### 3. **Community Hub**
- **Community Feed**: Displays "Public" projects shared by other users to foster transparency and shared learning.
- **Project Details**: View risk scores and summaries of community projects.

### 4. **Profile & Credits System**
- **User Profile**: Displays user details and avatar.
- **Credits**: Users start with 5 free credits. 1 credit deducted per analysis.
- **Pricing**: Interface to "purchase" credit packs.

## üì¶ Deployment

This project includes a `vercel.json` configuration and is ready to deploy on Vercel.

**IMPORTANT NOTE ON DATA PERSISTENCE:**
This application currently uses a file-based database (`data/db.json`). Vercel and similar serverless platforms have ephemeral file systems, meaning **data saved during a session will be lost** when the server sleeps or redeploys.

**For Production Use:**
1. Switch to a cloud database (e.g., Vercel Postgres, MongoDB Atlas, Supabase).
2. Update `lib/storage.ts` to connect to the external database.
